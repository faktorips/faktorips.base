This package provides the migration strategies for Faktor-IPS since version 3.2.

The new migration strategy does no longer needs any naming conventions for the classes. To find the migration easily, it is best practice to name the migration Migration_[Version]. There is also no need of creating a migration strategy for every version! But the migrations still extends from org.faktorips.devtools.core.model.versionmanager.AbstractIpsProjectMigrationOperation or even more precicely from org.faktorips.devtools.core.internal.migration.DefaultMigration.

If you need a migration strategy for a new Faktor-IPS version you have to create the migration strategy (best you do in this package) AND you have to register the migration strategy by extending the extension point org.faktorips.devtools.core.ipsMigrationOperation. In this extension you have to specify the migration strategy class next to the target version this strategy is responsible for. This target version sould be the same as the migration strategy return by the method getTargetVersion(). However it is not necessary to specify the exact version. As far the strategy is not responisble for a special qualifier version you could specify the version number without the qualifier.

E.g. the Version strategy is resposible to migrate from version 3.1.x to verion 3.3.0.rc1: you do not need the qualifier rc1, target version is 3.3.0.
If you need a migration from 3.1.0(.rc1) to 3.1.0.rfinal (very bad practice!): you need the qualifier, target version is 3.1.0.rfinal.
Remember that qualifiers are arranged alphabetically.
