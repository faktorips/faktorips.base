Hinweis

Nach dem Hinzufügen eines Vertragsteils kann es sein, dass der hinzugefügte Vertragsteil wiederum weitere Vertragsteile benötigt. Der Editor wird in diesem Fall die Test Datenstruktur automatisch erweitern, falls dies möglich ist. So kann es passieren, dass beim Hinzufügen eines einzelnen Vertragsteils ein ganzer Teilbaum eingehängt wird.

Die Test Datenstruktur wird nur automatisch erweitert, wenn der hinzugefügte Vertragsteil durch einen Produktbaustein konfiguriert ist. Der Automatismus funktioniert nach folgendem Schema:

  1. Zunächst wird durch den zugrunde liegenden Testfalltyp ermittelt, welche Vertragsteile wie oft an dem neuen Vertragsteil angehängt werden müssen. Die zugehörige Modellbeziehung an der Vertragsteilklasse muss zudem eine Komposition sein.

  2. Für alle Vertragsteile, die mindestens einmal angehängt werden müssen, wird nach möglichen Produktbausteinen gesucht. Wird genau ein Produktbaustein gefunden, so wird der Vertragsteil so oft hinzugefügt, wie es der Testfalltyp vorgibt. Den neuen Vertragsteilen wird der entsprechende Produktbaustein zugewiesen.

  3. Für alle Vertragsteile, die optional sind, oder für die kein eindeutiger Produktbaustein gefunden wurde, werden die Beziehungen des Eltern-Produktbausteins untersucht. Gibt es in irgendeiner Anpassungsstufe dieses Eltern-Produktbausteins eine Beziehung zu einem Produktbaustein, dessen Vertragsteilklasse der des gesuchten Vertragsteils entspricht, so wird diese Beziehung näher untersucht. Ist die Beziehung obligatorisch (Minimum-Kardinalität > 0), so werden so viele Vertragsteile hinzugefügt, wie es der Minimum-Kardinalität entspricht. Jedem Vertragsteil wird der Ziel-Produktbaustein dieser Beziehung zugewiesen.
